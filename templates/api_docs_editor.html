Format guide — how to write endpoint blocks 
::: endpoint
method: GET | POST | PUT | DELETE | PATCH
path: /resource/{id}
title: Short title
description: What this endpoint does.
auth: Bearer token   (optional override)

param: <name> | <location> | <type> | required/optional | description
  locations: query, body, path, header
  types: string, integer, boolean, array, object, number

response: 200 | Success
{
  "id": 1,
  "name": "example"
}

response: 400 | Bad Request
{"error": "Validation failed"}
:::

You can add free-form markdown above the first ::: endpoint block as project-level notes.
